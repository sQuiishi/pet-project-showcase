## 4. Бизнес-правила Flow

- Пользователь MUST иметь возможность выбрать действие `Login via Faceit`
- Система MUST начать обработку пользователя после получения внешнего идентификатора Faceit
- `FUNC-AUTH-RESOLVE-USER-ACCESS-001` MUST искать пользователя в локальной БД по внешнему идентификатору Faceit
- Если пользователь не найден, `FUNC-AUTH-RESOLVE-USER-ACCESS-001` MUST создать нового пользователя
- Для нового пользователя система MUST установить `is_onboarding_completed = N`
- Для нового пользователя система MUST получить данные профиля из Faceit и сохранить их в локальном профиле
- Если `is_onboarding_completed = N`, система MUST перенаправить пользователя на onboarding
- Если `is_onboarding_completed = Y`, система MUST перенаправить пользователя в market
- Пользователь MUST иметь возможность завершить onboarding
- `FUNC-ONBOARDING-COMPLETE-001` MUST сохранить введённые onboarding-данные
- `FUNC-ONBOARDING-COMPLETE-001` MUST установить `is_onboarding_completed = Y`
- После успешного завершения onboarding система MUST перенаправить пользователя в market
