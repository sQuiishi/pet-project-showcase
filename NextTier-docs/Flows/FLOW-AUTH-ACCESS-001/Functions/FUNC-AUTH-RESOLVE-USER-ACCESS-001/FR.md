# ID FUNC-AUTH-RESOLVE-USER-ACCESS-001

## 1. Description

Функция предназначена для обработки авторизованного пользователя в системе после получения его внешнего идентификатора.  
Система должна выполнить поиск пользователя по Faceit_ID, при необходимости создать нового пользователя, получить и сохранить базовые данные профиля, проверить статус прохождения onboarding, а затем вернуть результат, необходимый для определения дальнейшего маршрута пользователя.

## 2. Trigger (Когда выполняется)

Функция выполняется, когда:

- система получает запрос на обработку авторизованного пользователя;
- пользователь повторно входит в систему;
- пользователь возвращается в систему после незавершённого onboarding;
- пользователь повторно входит в систему после завершённого onboarding.

## 3. Preconditions (Предусловия)

- Faceit User ID передан в систему;
- система имеет доступ к данным профиля пользователя;
- в системе доступна возможность сохранения нового пользователя;
- в системе доступна возможность чтения статуса onboarding.

## 4. Functional Behavior
### Что может пользователь

- пользователь может инициировать вход в систему;
- пользователь может повторно войти в систему;
- пользователь может продолжить onboarding после повторного входа;
- пользователь может войти в систему после ранее завершённого onboarding.

### Что должна делать система

- система должна принимать запрос на обработку пользователя;
- система должна принимать `Faceit_ID`;
- система должна выполнять поиск пользователя по `Faceit_ID`;
- система должна проверять наличие пользователя в системе;
- если пользователь не найден, система должна создавать нового пользователя;
- если пользователь не найден, система должна сохранять `Faceit_ID` в профиле пользователя;
- если пользователь не найден, система должна получать базовые данные профиля пользователя;
- если пользователь не найден, система должна сохранять базовые данные профиля пользователя;
- если пользователь не найден, система должна устанавливать `is_onboarding_completed = N`;
- если пользователь не найден, система должна устанавливать `status = Draft`;
- если пользователь не найден, система должна определять маршрут перехода на onboarding;
- если пользователь найден, система должна проверять значение `is_onboarding_completed`;
- если пользователь найден и `is_onboarding_completed = N`, система должна определять маршрут перехода на onboarding;
- если пользователь найден и `is_onboarding_completed = Y`, система должна определять маршрут перехода на market;
- система должна возвращать маршрут дальнейшего перехода пользователя;
- система должна возвращать ошибку валидации, если `Faceit_ID` отсутствует или имеет некорректный формат;
- система должна возвращать внутреннюю ошибку, если обработка пользователя не может быть выполнена.
  
## 5. Postconditions (Результат выполнения)

### Успешное выполнение:

- пользователь найден или создан в системе;
- для нового пользователя создан профиль;
- для нового пользователя сохранён Faceit_ID;
- для нового пользователя сохранены базовые данные профиля;
- определены значения `is_onboarding_completed` и status;
- определён маршрут дальнейшего перехода пользователя;
- система вернула данные, необходимые для редиректа пользователя.

### Неуспешное выполнение:

- система не завершает обработку пользователя;
- система не возвращает маршрут перехода;
- система возвращает ошибку валидации или внутреннюю ошибку.
