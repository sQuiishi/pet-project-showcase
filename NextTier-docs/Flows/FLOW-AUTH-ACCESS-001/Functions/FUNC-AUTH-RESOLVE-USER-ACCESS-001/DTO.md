# DTO-AUTH-RESOLVE-USER-ACCESS-001
## 1. Общая идея документа

| Name                 | Description                                                                   |
| -------------------- | ----------------------------------------------------------------------------- |
| ID                   | DTO-AUTH-RESOLVE-USER-ACCESS-001                                              |
| Назначение документа | описывает входные, системные и выходные данные функции обработки пользователя |


## 2. Input Data — Входные данные

Назначение: Данные, которые передаются системой-клиентом для выполнения функции.

| Data Element | Назначение                         | Обязательное | Может быть пустым | Правила / ограничения | Пример    |
| ------------ | ---------------------------------- | ------------ | ----------------- | --------------------- | --------- |
| faceit_id    | внешний идентификатор пользователя | Да           | Нет               | string                | abcd-1234 |




## 3. System Data — Данные, формируемые системой

Назначение: Данные, которые не передаются извне, а определяются системой в процессе выполнения функции.

| Data Element            | Назначение                    | Когда формируется                         | Возможные значения  | Пример     |
| ----------------------- | ----------------------------- | ----------------------------------------- | ------------------- | ---------- |
| user                    | сущность пользователя         | при поиске / создании                     | object              | {...}      |
| is_onboarding_completed | статус прохождения onboarding | после получения пользователя              | Y / N               | N          |
| status                  | статус профиля пользователя   | после получения или создания пользователя | Draft / Active      | Draft      |
| next_route              | маршрут перехода пользователя | перед возвратом результата                | onboarding / market | onboarding |



#### user — контракт объекта пользователя, создаваемого системой

| Data Element            | Назначение                            | Тип           | Пример              |

| telegram                | telegram пользователя                 | link          | null                |

| primary_role            | основная роль                         | string        | null                |
| search_goal             | цель поиска                           | string        | null                |
| prime_time              | игровое время                         | string        | null                |
| is_marker_visible       | видимость игрока на маркете           | boolean       | false                |





## 4. Output Data — Данные, возвращаемые системой

Назначение: Данные, которые возвращаются после успешного выполнения функции.
!Данные возвращаются фронту, не пользователю!

| Data Element | Назначение            | Когда возвращается      | Возможные значения  | Пример     |
| ------------ | --------------------- | ----------------------- | ------------------- | ---------- |
| next_route   | маршрут для редиректа | при успешном выполнении | onboarding / market | onboarding |



## 5. Result Data — Результаты выполнения

| Outcome type | Result / Error name | Что возвращается / происходит         | Когда возникает                       |
| ------------ | ------------------- | ------------------------------------- | ------------------------------------- |
| Success      | user_processed      | пользователь найден или создан        | успешная обработка пользователя       |
| Error        | validation_error    | Система отклоняет запрос              | faceit_id отсутствует или некорректен |
| Error        | internal_error      | система не может завершить выполнение | внутренняя ошибка при обработке       |
