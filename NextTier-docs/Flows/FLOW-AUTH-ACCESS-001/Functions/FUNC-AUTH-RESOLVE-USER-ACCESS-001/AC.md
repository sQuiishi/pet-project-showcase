# AC-AUTH-RESOLVE-USER-ACCESS-001

## 1. Общая информация о критериях приёмки

| Name     | Description                                                                          |
| -------- | ------------------------------------------------------------------------------------ |
| ID       | AC-AUTH-RESOLVE-USER-ACCESS-001                                                      |
| Описание | критерии приёмки для функции обработки пользователя и определения маршрута редиректа |

## 2. Таблица сценариев

| AC ID | Сценарий                                                     | Given (Начальное состояние)                                                                                                            | When (Действие)                                  | Then (Ожидаемый результат)                                                                                                                                                                                                                                                   |
| ----- | ------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| AC-1  | создание нового пользователя при отсутствии в БД             | в системе передан валидный `faceit_id`; пользователь с таким `faceit_id` отсутствует в БД                                              | система выполняет функцию обработки пользователя | система MUST создать нового пользователя; система MUST сохранить `faceit_id`; система MUST сохранить базовые данные профиля; система MUST установить `is_onboarding_completed = N`; система MUST установить `status = Draft`; система MUST вернуть `next_route = onboarding` |
| AC-2  | редирект найденного пользователя на onboarding               | в системе передан валидный `faceit_id`; пользователь с таким `faceit_id` существует в БД; у пользователя `is_onboarding_completed = N` | система выполняет функцию обработки пользователя | система MUST NOT создавать нового пользователя; система MUST вернуть `next_route = onboarding`                                                                                                                                                                               |
| AC-3  | редирект найденного пользователя на market                   | в системе передан валидный `faceit_id`; пользователь с таким `faceit_id` существует в БД; у пользователя `is_onboarding_completed = Y` | система выполняет функцию обработки пользователя | система MUST NOT создавать нового пользователя; система MUST вернуть `next_route = market`                                                                                                                                                                                   |
| AC-4  | ошибка валидации при отсутствии или некорректном `faceit_id` | `faceit_id` отсутствует или некорректен                                                                                                | система выполняет функцию                        | система MUST вернуть `validation_error`                                                                                                                                                                                                                                      |
| AC-5  | внутренняя ошибка выполнения функции                         | во время выполнения произошла внутренняя ошибка                                                                                        | система выполняет функцию                        | система MUST вернуть `internal_error` и MUST NOT возвращать `next_route`                                                                                                                                                                                                     |


- В критериях приемки оставлены только те сценарии, которые отражают поведение, реализуемое разработчиком.
- Клиентские ошибки, автоматические ошибки сервера и внутренние технические сбои, не требующие отдельной ручной обработки, в критерии не включены.
