# ID FUNC-ONBOARDING-COMPLETE-001

## 1. Description

Функция предназначена для завершения onboarding пользователя в системе.  
Система должна принять заполненную форму onboarding, проверить обязательные поля и согласие на обработку данных, сохранить данные профиля пользователя, установить признак завершения onboarding, обновить статус профиля пользователя, а затем вернуть результат, необходимый для определения дальнейшего маршрута пользователя.

## 2. Trigger (Когда выполняется)

Функция выполняется, когда:

- пользователь отправляет форму onboarding.

## 3. Preconditions (Предусловия)

- пользователь существует в системе;

## 4. Functional Behavior
### Что может пользователь

- пользователь должен заполнить обязательные поля onboarding;
- пользователь может заполнить необязательное поле игрового времени;
- пользователь должен отправить форму onboarding.

### Что должна делать система

- система должна принимать запрос на завершение onboarding;
- система должна принимать данные формы onboarding;
- система должна проверять наличие обязательных полей:
    - `telegram`
    - `birth_date`
    - `primary_role`
    - `search_goal`
- система должна проверять корректность значений переданных полей;
- система должна сохранять значение `telegram` в профиле пользователя;
- допускается ввод telegram в формате `nickname` или `@nickname`
- система должна валидировать значение telegram в ссылку формата t.me/...
- система должна сохранять значение `birth_date` в профиле пользователя;
- система должна сохранять значение `primary_role` в профиле пользователя;
- система должна сохранять значение `search_goal` в профиле пользователя;
- если передано значение `prime_time`, система должна сохранять значение `prime_time` в профиле пользователя;
- если не передано значение `prime_time`, система должна установить значение `prime_time` не указано;
- система должна устанавливать `is_onboarding_completed = Y`;
- система должна устанавливать `status = Active`;
- система должна установить is_market_visible = N/Y
- система должна определять маршрут перехода пользователя на market;
- система должна возвращать маршрут дальнейшего перехода пользователя;
- система должна возвращать ошибку валидации, если обязательные поля отсутствуют, переданы некорректно или согласие на обработку данных не установлено;
- система должна возвращать внутреннюю ошибку, если завершение onboarding не может быть выполнено.
  
## 5. Postconditions (Результат выполнения)

### Успешное выполнение:

- данные onboarding сохранены в профиле пользователя;
- для пользователя установлено значение `is_onboarding_completed = Y`;
- для пользователя установлено значение `status = Active`;
- определён маршрут дальнейшего перехода пользователя;
- система вернула данные, необходимые для редиректа пользователя.

### Неуспешное выполнение:

- система не завершает onboarding пользователя;
- система не устанавливает признак завершения onboarding;
- система не возвращает маршрут перехода;
- система возвращает ошибку валидации или внутреннюю ошибку.
