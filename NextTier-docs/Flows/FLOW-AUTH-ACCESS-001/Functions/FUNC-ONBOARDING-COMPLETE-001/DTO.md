# DTO-ONBOARDING-COMPLETE-001
## 1. Общая идея документа

| Name                 | Description                                                                  |
| -------------------- | ---------------------------------------------------------------------------- |
| ID                   | DTO-ONBOARDING-COMPLETE-001                                                  |
| Назначение документа | описывает входные, системные и выходные данные функции завершения onboarding |


## 2. Input Data — Входные данные

Назначение: Данные, которые передаются системой-клиентом для выполнения функции.

| Data Element          | Назначение                   | Обязательное | Может быть пустым | Правила / ограничения                                           | Пример           |
| --------------------- | ---------------------------- | ------------ | ----------------- | --------------------------------------------------------------- | ---------------- |
| telegram              | контакт пользователя         | Да           | Нет               | string                                                          | @bro             |
| birth_date            | дата рождения                | Да           | Нет               | date                                                            | 2004-06-15       |
| primary_role          | основная роль                | Да           | Нет               | enum(IGL, AWPer, Riffler, Support, Entry fragger, Lurker, Flex) | awper            |
| search_goal           | search_goal                  | Да           | Нет               | enum (looking_for_players / looking_for_team / not_searching)   | looking_for_team |
| prime_time            | игровое время                | Нет          | Да                | time                                                            | 17.30-18.30      |
| personal_data_consent | согласие на обработку данных | Да           | Нет               | Y / N                                                           | Y                |

## 3. System Data — Данные, формируемые системой

Назначение: Данные, которые не передаются извне, а определяются системой в процессе выполнения функции.

| Data Element            | Назначение                           | Когда формируется                                                                     | Возможные значения | Пример   |
| ----------------------- | ------------------------------------ | ------------------------------------------------------------------------------------- | ------------------ | -------- |
| user                    | сущность пользователя                | при обновлении профиля                                                                | object             | {...}    |
| telegram                | нормализованное значение telegram    | при обработке входных данных                                                          | link               | t.me/bro |
| is_onboarding_completed | статус onboarding                    | после сохранения                                                                      | Y / N              | Y        |
| status                  | статус профиля пользователя          | после сохранения                                                                      | Draft / Active     | Active   |
| is_market_visible       | видимость карточки игрока на маркете | если игрок выбрал looking_for_team - Y<br>looking_for_players и not_searching - N<br> | Y/N                | Y        |
| next_route              | маршрут перехода пользователя        | перед возвратом результата                                                            | market             | market   |
#### Данные обновляемые системой в объекте user
| Data Element            | Назначение                    | Тип     | Обновляем на     |
| ----------------------- | ----------------------------- | ------- | ---------------- |
| is_onboarding_completed | признак завершения onboarding | string  | Y                |
| status                  | статус профиля пользователя   | string  | Active           |
| telegram                | telegram пользователя         | link    | t.me/bro         |
| birth_date              | birth_date                    | date    | 27.06.2005       |
| primary_role            | основная роль                 | string  | Awper            |
| search_goal             | цель поиска                   | string  | look_for_team    |
| prime_time              | игровое время                 | string  | null/17.00-18.00 |
| is_market_visible       | карточка игрока на маркете    | boolean | Y                |


## 4. Output Data — Данные, возвращаемые системой

Назначение: Данные, которые возвращаются после успешного выполнения функции.
!Данные возвращаются фронту, не пользователю!

| Data Element | Назначение            | Когда возвращается      | Возможные значения | Пример |
| ------------ | --------------------- | ----------------------- | ------------------ | ------ |
| next_route   | маршрут для редиректа | при успешном выполнении | market             | market |



## 5. Result Data — Результаты выполнения

| Outcome type | Result / Error name  | Что возвращается / происходит         | Когда возникает                               |
| ------------ | -------------------- | ------------------------------------- | --------------------------------------------- |
| Success      | onboarding_completed | onboarding успешно завершён           | все данные валидны и сохранены                |
| Error        | validation_error     | Система отклоняет запрос              | обязательные поля отсутствуют или некорректны |
| Error        | internal_error       | система не может завершить выполнение | внутренняя ошибка при обработке               |
