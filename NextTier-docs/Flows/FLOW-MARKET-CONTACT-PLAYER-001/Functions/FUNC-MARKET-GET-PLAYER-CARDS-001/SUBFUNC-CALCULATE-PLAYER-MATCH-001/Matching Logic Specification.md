## 2. Назначение алгоритма
Алгоритм рассчитывает процент соответствия между текущим пользователем и карточкой игрока для отображения на маркете.

Результат используется только в UI и не сохраняется в БД.
## 3. Место в процессе

- `FUNC-MARKET-GET-PLAYER-CARDS-001` получает игроков из БД
- Для каждой карточки вызывается алгоритм расчета
- Алгоритм возвращает процент соответствия
- Значение добавляется в данные карточки
- Карточки передаются в UI
## 4. Входные данные

Алгоритм получает:
#### Данные текущего пользователя:
- primary_role
- elo
- prime_time (временной диапазон)
- age
#### Данные игрока:
- primary_role
- elo
- prime_time
- age
## 5. Логика расчета

Итоговый процент рассчитывается на основе 4 критериев:

- роль (самый приоритетный критерий)
- elo
- игровое время
- возраст (минимальный вес)

Итоговый процент = сумма баллов по критериям.

## 6. Правила расчета

| Метрика    | Процент |
| ---------- | ------- |
| elo_score  | 39%     |
| time_score | 33%     |
| role_score | 20%     |
| age_score  | 8%      |
Итог: Эло+Игровое время + Роль + Возраст = 100%

#### Итоговая формула:

reccomend_market =  (elo_score * high__elo_coefficient or low__elo_coefficient) + (time_score * time_coefficient) + (role_score * role_coefficient) + (age_score * age_coefficient)
