# ID FUNC-MARKET-GET-PLAYER-CARDS-001

## 1. Description

Функция предназначена для получения карточек игроков, доступных для отображения на странице Market.  
Система должна получить карточки только опубликованных игроков и, если переданы параметры фильтрации и/или сортировки, применить их к результату.

## 2. Trigger (Когда выполняется)

Функция выполняется, когда:

- пользователь открывает страницу Market;
- пользователь изменяет параметры фильтрации;
- пользователь изменяет параметры сортировки;
- пользователь сбрасывает параметры фильтрации и/или сортировки.

## 3. Preconditions (Предусловия)

- пользователь находится на странице Market;
- пользователь авторизован в системе;
- система имеет доступ к данным игроков.

## 4. Functional Behavior
### Что может пользователь

- пользователь может открыть страницу Market;
- пользователь может запросить список карточек игроков без параметров фильтрации и сортировки;
- пользователь может передать параметры фильтрации;
- пользователь может передать параметры сортировки;
- пользователь может передать одновременно параметры фильтрации и сортировки;
- пользователь может сбросить ранее выбранные параметры фильтрации и сортировки.

### Что должна делать система

- система должна принимать запрос на получение карточек игроков для страницы Market;
- система должна выбирать только игроков, у которых `is_market_visible=true`;
- система не должна включать в результат игроков, у которых is_market_visible=false`;
- система должна получать данные игроков, необходимые для формирования карточек Market;
- система должна формировать список карточек игроков;
- система должна применять фильтрацию, если переданы параметры фильтрации;
- система должна применять сортировку, если переданы параметры сортировки;
- система должна сначала применять фильтрацию, а затем сортировку, если переданы оба типа параметров;
- система должна возвращать итоговый список карточек игроков;
- система должна возвращать количество карточек в результате;
- система должна возвращать успешный результат, если запрос обработан успешно;
- система должна возвращать пустой список, если подходящие игроки не найдены;
- система должна возвращать ошибку валидации, если переданы некорректные параметры запроса.
  
## 5. Postconditions (Результат выполнения)

### Успешное выполнение

- система сформировала список `market_player_cards`;
- система вернула только игроков с `is_market_visible = true`;
- если были переданы параметры фильтрации — они применены к результату;
- если были переданы параметры сортировки — они применены к результату;
- если были переданы и фильтрация, и сортировка — сначала применена фильтрация, затем сортировка;
- система вернула `players_count`, соответствующий количеству карточек;
- система вернула `current_page` и `total_pages`
- каждая карточка соответствует контракту `market_player_card`

### Частично успешное (empty)

- система вернула `market_player_cards = []`;
- система вернула `players_count = 0`;
- система MAY вернуть `message` для отображения пользователю.
### Неуспешное выполнение

- система не возвращает список карточек игроков;
- система возвращает `validation_error`, если параметры некорректны;
- система возвращает `internal_error`, если выполнение невозможно.
