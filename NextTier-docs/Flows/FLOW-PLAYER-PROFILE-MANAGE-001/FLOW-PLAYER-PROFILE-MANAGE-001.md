## 4. Бизнес-правила Flow

- Пользователь MUST быть авторизован для редактирования профиля.
- Пользователь MUST иметь возможность открыть форму редактирования профиля.
- Система MUST отобразить в форме текущие известные данные профиля.
- Пользователь MAY изменить `telegram`, `prime_time`, `primary_role` и `avatar`.
- `FUNC-PLAYER-PROFILE-UPDATE-001` MUST определить профиль авторизованного пользователя.
- `FUNC-PLAYER-PROFILE-UPDATE-001` MUST валидировать переданные данные.
- Система MUST принимать `telegram` в формате `nickname`, `@nickname`
- Система MUST нормализовать `telegram` в формат `t.me/...`.
- Если передан новый `avatar`, система MUST загрузить файл и сформировать `avatar_url`.
- Если входные данные корректны, `FUNC-PLAYER-PROFILE-UPDATE-001` MUST обновить профиль в БД.
- Если входные данные некорректны, система MUST вернуть `validation_error` и MUST отобразить пользователю ошибки в полях.
- Если пользователь пытается изменить недоступную ему запись, система MUST вернуть `forbidden`.
- Чтобы удалить `prime_time`, пользователь MUST передать `prime_time = NULL`.
- После успешного обновления система MUST вернуть пользователю обновлённые данные профиля.
